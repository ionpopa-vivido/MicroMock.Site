Scopri MicroMock

Cos'è Micromock

MicroMock ti consente di configurare ed eseguire microservizi mock per simulare lo scambio di messaggi tra microservizi, tutto tramite una semplice interfaccia grafica e senza scrivere una linea di codice. Elimina le dipendenze con altri team durante il test del tuo microservizio e parla con l’assistente AI in linguaggio naturale per velocizzare la configurazione e l’avvio dei tuoi microservizi mock. MicroMock supporta vari message broker come Kafka, RabbitMQ, Azure Service Bus e API endpoint.

Caso d'uso: Generazione ordine

E’ in corso lo sviluppo di tre microservizi che si occuperanno di generare e finalizzare un ordine: OrderService, InventoryService e PaymentService. Lo sviluppo potrebbe essere in carico a tre team diversi, tre persone diverse dello stesso team o in carico ad un’unica persona. Poiché i processi dipendono dagli esiti dei vari microservizi, lo sviluppatore deve simulare lo scambio di messaggi per testare tutte le casistiche. Nel nostro esempio, il TEAM_A è responsabile dello sviluppo di OrderService, mentre gli altri due microservizi non sono ancora disponibili per I test di scambio di messaggi.

Il supporto dell'intelligenza articifiale

AI Hero Assistant

Per semplificare e ridurre al minimo il tempo di configurazione, MicroMock dispone di un asssistente integrato basato sull'intelligenza articifiale.
I prompt di sistema sono scritti direttamente dal team di MicroMock, garantendo un alto grado di affidabilità nelle risposte fornite dal servizio AI.
L'utente, in base alle proprie esigenze, può delegare all'AI, tramite comandi in linguaggio naturale, le seguenti attività:

  • Personalizzazione

    Costruzione di regole per la scelta della risposta.

  • Mapping

    Mapping automatico dei valori tra JSON in ingresso e quello in uscita

  • Template

    Creazione di template JSON complessi come risposta

  • Congruità del modello

    Verifica automatica della congruità tra JSON in ingresso e quello in uscita.

  • Dati simulati

    Generazione di dati fake complessi.

Sicurezza dei dati

Quando si utilizza l'AI, è fondamentale prestare attenzione ai dati inviati a terze parti. MicroMock ha scelto di utilizzare l'AI di Microsoft tramite i servizi di Azure OpenAI, che garantiscono la sicurezza dei dati.
Se l'utente decide di utilizzare l'assistente AI, MicroMock invierà esclusivamente i dati presenti nei JSON o nei template mock (comprese le regole di mapping o selezione delle risposte). Credenziali o certificati di collegamento dei message broker non verranno mai inviati ad Azure OpenAI.
Inoltre, MicroMock non raccoglie alcun tipo di dato, né a livello statistico né per finalità di controllo.

Panoramica della struttura

Architettura

MicroMock UI

L'interfaccia utente è realizzata in Angular 18 con un tema personalizzato.
Permette la gestione completa di MicroMock Service e le caratteristiche principali sono:

  • Gestione di workspaces per lavorare su progetti diversi.
  • Modifica in tempo reale delle configurazioni.
  • Possibilità di utilizzo con tre modalità: installazione locale (tramite Electron), Docker, deploy su server (ad esempio Kubernetes).

MicroMock Service

Sviluppato in .NET 8 utilizzando il template Web API, che consente l'interazione tramite REST con l'interfaccia utente.
Gestisce il ciclo di vita dei mock service per un determinato workspace, includendo:

  • Avvio e arresto selettivo dei microservizi mock.
  • Attività di consumer per lettura dei messaggi dal message broker.
  • Attività di producer e gestione delle regole per la generazione dei messaggi.

Installazione e deploy

MicroMock offre massima flessibilità nell'installazione, adattandosi alle esigenze specifiche:

  • Installazione locale tramite framework Electron, supportando lo sviluppatore nei test durante la fase di sviluppo. Un unico pacchetto gestisce sia l'interfaccia utente sia il servizio di background.
  • Deploy su Docker di MicroMock UI e Service, senza necessità di installazione diretta sulla macchina.
  • Deploy come container su server di DEV o QA, con esempi di deploy disponibili per Kubernetes.
